# Security Policy

## Supported Versions

The following versions of the Happy Corner project are currently supported with security updates:

| Version | Supported |
|--------|-----------|
| Main / Latest | Yes |
| Older versions | No |

Only the latest version available in the main branch receives security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

Do not open a public issue to report security problems.

Instead, follow these steps:

1. Clearly describe the vulnerability.
2. Include steps to reproduce the issue if possible.
3. Explain the potential impact.
4. Send the report privately to the project maintainer.

Contact:
- Instagram: @happycornerca
- E-mail:security@happycorner.lol, somos@happycorner.lol (PLEASE FORWARD TO BOTH OF THEM!!)
- Or through a private communication channel agreed with the maintainer.

You can expect an initial response within a reasonable timeframe.

---

## Security Best Practices

This project follows basic security practices, including:
- Limited access to sensitive data
- No storage of unnecessary personal information
- Regular review of public content and configurations

As this is a student-led project, security measures may evolve over time.

---

## Responsible Disclosure

Please allow time for the issue to be reviewed and resolved before sharing details publicly.

Responsible disclosure helps keep users and the project safe.
